DBMS (DATABASE MANAGEMENT SYSTEM):


Database Management Systems (DBMS) are software systems used to store, retrieve, and run queries on data. A DBMS serves as an interface between an end-user and a database, allowing users to create, read, update, and delete data in the database.

DBMS FEATURES:


The features of DBMS software are as follows:

  • Minimum repetition and redundancy: In a database there are high chances of data duplication due to the presence of many users. DBMS resolves this issue by building a repository where all the data are stored and can be accessed by multiple users.
  • Databases of big corporations or companies can only be managed by a database management system. Database management system offers data protection and features like recovery and backup for easy recording and retrieval of data.
  • Enhanced security: Database management systems offer enhanced security by restricting public access. It can only be accessed by a database administrator or the concerned head of the department.
  • Multiple user access: Database is designed in such a way that it can be accessed by multiple users who can work on different sections of the databases at a time without interrupting the flow of work.
  • Provides multiple views of the data: Database management system allows the users to access multiple views of the data present in the database. For example, an HR personnel can check the database for processing the data of the employee, while at the same time an employee can use the database to access resources related to payslip, attendance, etc.

Definitions:

  • Data: Raw facts and figures.
  • Information: Processed data that is meaningful.
  • Metadata: Data about data.
  • Data Dictionary: A repository of information about data elements.
  • Table: A collection of related data held in a structured format within a database.
  • Tuples: Rows in a table representing a single data item.
  • Fields: Columns in a table, representing a category of data.
  • Domain: The set of permissible values a field can have.
  • Constraints: Rules applied to data to ensure its validity.
    1. Super Keys:The collection of all data including identifier, unique and non-unique values
    2. Primary Keys: A super key with no duplicate values
    3. Foreign Keys: A key that references the primary key of another
    4. Candidate Keys:Collection of all unique data . It includes primary keys and alternate keys combined.
    5. Alternate Keys:Collection of all unique data which are in Candidate keys except the primary key.
    6. Composite Keys:Combination of primary,candidate,alternate keys with other keys.
    7. Artificial Keys:New key or data added during the normalization of table.
